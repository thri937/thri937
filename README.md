<summary><b>🛡️ SentinAI — Multi-Model Ensemble Web Application Firewall</b></summary>
<br/>

An ML-based Web Application Firewall detecting **SQL Injection and XSS attacks** in real time, using TF-IDF vectorization and an ensemble of 11 machine learning models.

| Category | Details |
|---|---|

| **Security** | Real-time SQLi/XSS attack detection with SOC-style monitoring |
| **Impact** | Built with a live Streamlit SOC dashboard for analyst visibility |
| **Repository** | [View Repository](https://github.com/thri937/sentinai) |

<summary><b>📍 GradEcho — Geo-Spatial Campus Memory Platform (Android)</b></summary>
<br/>

A Kotlin Android application enabling **location-based digital memory drops** — students leave notes/media locked to a physical location that others can only unlock by being physically present.

| Category | Details |
|---|---|
| **Stack** | Kotlin, Jetpack Compose, Firebase Firestore, Firebase Cloud Storage, OSMDroid |
| **Scale** | Campus-wide geo-tagged content platform |
| **Performance** | Precise proximity detection via FusedLocationProviderClient |
| **Security** | Geo-locking requiring physical proximity within 50m to unlock content |
| **Impact** | Fully serverless backend architecture on Firebase |
| **Repository** | [View Repository](https://github.com/thri937/gradecho) |

The core engineering challenge was reliable, battery-efficient proximity gating — solved using FusedLocationProviderClient with a 50-meter geofence, backed by a serverless Firestore + Cloud Storage architecture for scale without infrastructure overhead.

</details>

<details>
<summary><b>🔏 Commitment Scheme Based Exam Submission Verifier</b></summary>
<br/>

A secure exam submission system built on the **Pedersen Commitment Scheme**, designed to cryptographically detect tampered submissions.

| Category | Details |
|---|---|
| **Stack** | Applied cryptography, Python, verification portal |
| **Scale** | Submission integrity verification system |
| **Performance** | Deterministic tamper detection via commitment verification |
| **Security** | Pedersen Commitment: `C = g^m · h^r mod p` |
| **Impact** | Verifiable, tamper-evident academic submission workflow |
| **Repository** | [View Repository](https://github.com/thri937/commitment-exam-verifier) |

Implements the Pedersen commitment formula directly to bind a submission to a hidden value that can later be verified without revealing it prematurely — giving cryptographic, rather than procedural, guarantees against tampering.

</details>
